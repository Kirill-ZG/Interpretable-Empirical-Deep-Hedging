# Artifact Manifest

This repository keeps the paper reproducible in two layers.

1. `scripts/make_paper_figures.py` regenerates manuscript figures and LaTeX
   tables from saved paper-input artifacts.
2. The scripts below document the intended generator for each saved artifact
   class.

The paper-input artifacts live under:

```text
reproducibility_artifacts/paper_inputs/
```

The first layer was verified by running:

```powershell
python scripts\make_paper_figures.py
```

For command-level notes, see
`reproducibility_artifacts/COMMANDS.md`.
Small provenance manifests copied from the original result folders are stored
under `reproducibility_artifacts/generation_manifests/`.
Symbolic-distillation fit and smoother manifests are additionally collected in
`reproducibility_artifacts/generation_manifests/symbolic_distillation/`.

Selected trained actor/scaler files are stored under seed-family subfolders in
`model/`; their manifest is
`reproducibility_artifacts/trained_models/selected_model_manifest.csv`.

Hall-of-Fame formula/equation artifacts are stored in
`reproducibility_artifacts/hall_of_fame_formulas/`.

## Second-Layer Status

The exact historical commands used to create every saved artifact were not
recorded.  The table above is therefore a provenance manifest rather than a
guaranteed execution log.

For publication reproducibility, the first layer is the essential layer: all
figures and tables can be regenerated from the saved inputs.  The second layer
is documented through reconstructed commands and manifests, but not publicly executable without licensed raw data.

## Size Note

The copied first-layer artifacts are small enough for a normal GitHub
repository: the largest required CSV is about 13.5 MB, and no copied artifact
approaches GitHub's 100 MB per-file limit.

The selected trained actors and scalers add about 7.1 MB.  The copied
Hall-of-Fame formula, equation, and validation-selection artifacts add less
than 0.9 MB.

The full raw seed-result directory is much larger and is not required for the
first layer because the compact seed summaries above are already saved.

## Large Output Policy

Large second-layer intermediates are intentionally not included.  This
repository keeps:

- first-layer inputs required by `scripts/make_paper_figures.py`;
- selected trained actors and scalers;
- formula/equation text for symbolic policies;
- compact provenance manifests.

It omits large rerunnable intermediates such as full HOF trade-step tables,
paired-episode bootstrap inputs, temporary yearly DATA_DIR folders, and full
checkpoint grids.  Those files can be regenerated by licensed data holders with
the commands in `reproducibility_artifacts/COMMANDS.md`.
